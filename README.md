# AI 주식 자동매매 플랫폼

> 상태: **계획 단계** (구현 미착수) · 개인용 1인 프로젝트

한국 주식 시장을 대상으로, AI가 백엔드에서 자동으로 매매 판단·집행을 수행하고 그 결과를 대시보드로 확인하는 개인용 자동매매 시스템이다.  
사용자가 직접 매매하지 않고, AI의 판단과 코드의 검증을 거쳐 주문이 나간다. "그래도 되나?" 생각이 드는 만큼 당연히 수익도 중요하지만 안전이 또다른 큰 목표이다.

## 무엇을 하는가

- **자동 매매**: 평일 정해진 시각(오전 10:00·오후 14:30)에 매매 사이클이 자동으로 돌며, AI가 종목을 분석해 매수·매도·보유 결정을 내린다.
- **이벤트 대응**: 정기 사이클 외 시간 동안은 30분 간격으로 데이터를 수집해 보유 종목에 한해서만 급변(공시·가격 급등락)이 발생하면 정기 사이클 이 실행되는 구조이다.
- **리스크 통제**: 손절·익절·포지션 한도·일일 손실 한도 등은 AI가 아니라 코드가 강제한다.
- **보고 대시보드**: 수익·포지션·매매 내역·결정 근거·학습 교훈을 시각화해서 보여준다.

## 핵심 설계 철학

이 프로젝트는 "더 똑똑한 모델"이 아니라 "더 나은 데이터와 워크플로우, 그리고 규율 있는 리스크 통제"에 가치를 둔다.

1. **LLM은 제안하고, 코드가 집행한다.** AI의 출력은 *제안 주문*일 뿐이며, 코드의 리스크 검증을 통과해야만 실제 주문이 나간다. 지표 계산·손익 계산·리스크 검증·주문 집행은 전부 코드가 담당한다.
2. **모델보다 데이터·워크플로우가 핵심이다.** 어떤 데이터를 어떻게 압축해 어느 에이전트에 보내고, 결정론적 코드로 어떻게 집행하느냐가 이 시스템의 핵심이다.
3. **프레임워크 없이 직접 조립한다.** LangChain 같은 오케스트레이션 프레임워크를 쓰지 않고, 평범한 파이썬 함수 + `asyncio`로 워크플로우를 직접 구현한다. 매매에선 "무엇을 보냈고 무엇을 받았나"가 1차 자료이므로 투명성·추적 가능성을 우선시 한다.
4. **경험으로 학습한다.** 모델 가중치를 학습시키지 않고, *결정 → 실제 손익 → 교훈 → 다음 결정에 주입*하는 피드백 루프로 같은 실수를 반복하지 않도록 한다.
5. **측정이 없으면 수익 또한 없다.** 단순 전략(매수 후 보유 등)과 거래비용까지 빼고 비교해 이겨야 비로소 가치가 있다고 할 수 있다. 모의투자·소액 실거래로 먼저 검증한다.

## 한 사이클의 흐름

코드가 후보·데이터·기억을 준비 → AI가 분석·반박·결정 → 코드가 검증·집행·기록하는 9단계로 실행된다.

1. **후보 선별** (코드) — 전 종목에서 매매 부적합 종목을 거르고 점수순 상위 N개 + 보유 종목을 묶어 워치리스트 구성
2. **데이터 수집** (코드) — 시세·수급·기술지표, 미국 증시·환율·금리, 이벤트 캘린더 적재
3. **기억 검색** (코드) — 과거 비슷한 상황의 교훈 인출
4. **분석 에이전트** (AI 5명 병렬) — 기술·수급·글로벌매크로·촉매·포트폴리오 전문가가 각자 의견서 작성
5. **베어케이스** (AI) — "악마의 변호인"이 의도적으로 반박
6. **결정 에이전트** (AI) — 신규 진입·보유 포지션 관리를 JSON 제안 주문으로 출력
7. **리스크 검증** (코드) — 잔고 정합성·하드룰·서킷브레이커 통과 검사
8. **주문 실행** (코드) — 증권사 API로 주문 송출 (손절은 미리 등록)
9. **기록** (코드) — 입력·분석·결정·체결을 전부 SQLite에 저장

사이클과 별개로, **회고 에이전트**가 장 마감 후 예측 대비 실제 손익을 비교해 교훈을 남기고, **대시보드**가 같은 데이터를 읽어 사용자에게 보여준다.

## 기술 스택


| 영역        | 선택                                           |
| --------- | -------------------------------------------- |
| 언어        | Python                                       |
| AI 호출     | `anthropic` 공식 SDK (Claude) + `asyncio`      |
| 증권사·시세    | 한국투자증권 KIS Developers API + `httpx`          |
| 글로벌·매크로   | `yfinance`, `fredapi`                        |
| 공시        | DART API                                     |
| 데이터 처리·지표 | `pandas`, `numpy`, `exchange_calendars`      |
| 스키마·설정    | `pydantic` v2, `pydantic-settings`           |
| 저장소       | `sqlite3` (단일 파일, 모든 프로세스의 공유점)              |
| 대시보드      | Streamlit + Plotly (향후 FastAPI + Next.js 검토) |
| 테스트       | `pytest` 계열                                  |


모의투자와 실전 투자는 설정값(도메인·계좌번호) 하나로 전환되며, 코드는 동일하게 동작한다.
