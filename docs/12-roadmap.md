## 12. 단계별 구현 로드맵

> 본 로드맵은 *무엇을 어떤 순서로 만들고, 각 단계를 무엇으로 합격 판정하는가*의 단일 소스다. End-to-End 검증 절차·합격증 경계(기능 vs 전략)는 10장(`10-evaluation.md`)과 정합한다.
>
> **읽는 법**: 각 Phase는 **목표 / 산출물 / 합격 게이트(다음 단계로 넘어가는 조건)**로 구성된다. 괄호 안 모듈명(`broker/kis_client.py` 등)의 *전체 리포 구조 단일 소스는 3-3(리포 구조)*이며, 본 로드맵은 *순서·게이트*를, 3-3은 *구조*를 규정한다. 구조 자체는 Phase 0 스캐폴딩에서 그대로 생성한다.
>
> **호스트 전제 (Phase 0 확정)**: Phase 0 선검증 결과 **NCP(네이버 클라우드) 서울 상시 서버**로 확정했다 — KIS 실전·모의 토큰 발급이 모두 통과(HTTP 200). 당초 1순위 GitHub Actions(무료)는 KIS가 해외 IP(러너=Azure)를 차단(ConnectTimeout)해 탈락(Oracle Cloud는 가입 자체가 막혀 미시도). 더 먼 목표는 집 맥미니(가정용 IP·완전 자가소유·WebSocket). 설계가 *호스트 비의존*(3.3)이라 호스트 전환 비용은 작다.

### 12.0 진행 원칙 (로드맵 전체 관통)

- **게이트 기반 진행**: 각 Phase는 다음으로 넘어가기 전 명시된 합격 기준을 통과해야 한다. 미달 시 진행 차단(특히 실자본 투입 게이트).
- **선검증 우선 (de-risk first)**: "이게 안 되면 나머지가 무의미한" 전제(호스트 접속·백테스트 데이터 가용성)를 *본격 코딩 전에* 테스트한다(Phase 0 선검증 게이트). 늦게 발견하면 그동안 만든 게 헛수고가 된다.
- **측정 없으면 실자본 없다**: 실거래는 *Phase 6 백테스트 방향성 게이트* + *Phase 7.5 전략 합격증*을 통과한 뒤에만. 모의(Phase 7) KPI는 기능 합격만, 전략 합격으로 쓰지 않는다(5.2·10-2 모드 갭).
- **결정론 먼저**: 리스크·사이징·청산(Phase 3)을 *결정 파이프라인(Phase 4)보다 먼저* 머지한다 — LLM 제안이 나오기 전에 그것을 검증·집행할 바닥이 있어야 한다.
- **비용 0 우선**: 무거운 파라미터 튜닝은 *비용 0인 코드 경로(A안)*에서 워크포워드로 끝내고, LLM 호출은 최소(린 구성·3자 비교 절차 10-3).
- **운영 신뢰성은 원칙을 Phase 0에 박고 구현은 단계적**: 11장 18항목의 *원칙*은 Phase 0에 모두 명문화돼 있어야 하나, *구현*은 11-2.16 우선순위대로 Phase 0→1→2~3→7에 나눠 박는다.

### Phase 0 — 선검증 게이트 + 스캐폴딩

목표: ① 프로젝트가 *현 전제(호스트·데이터)*로 가능한지를 코드 본격 착수 전에 판명하고 ② 모든 후속 Phase가 딛고 설 리포·설정·스키마·운영 바닥을 깐다.

**0-A. 선검증 게이트 (Pre-flight — 최소 코드·결정 위주, *여기서 막히면 전제부터 재설계*)**

- **0-A.1 API 발급 (전부 무료)**: KIS 증권계좌 개설(입금 0원 OK) → App Key/Secret 발급 → 모의투자 부가 신청. 부수로 FRED·DART·네이버 검색 API 키 발급. 시크릿은 *처음부터* `.env`(로컬·NCP 서버)에만 두고 리포에 올리지 않는다(11-2.5).
- **0-A.2 호스트 접속 게이트 (P0-1) — *차단 게이트***: 후보 호스트(GitHub Actions / Oracle Cloud 서울 / 집 기기)에서 *KIS 인증 + 잔고 조회*를 실제 호출해 **해외 IP 차단 여부를 판명**한다. KIS는 Tier0 전부(계좌·시세·수급·공매도·대차)와 4b의 단일 출처라, 막히면 데이터·매매·잔고조회가 *통째로 정지*한다. 결과로 **현 단계 호스트를 확정**한다. **실행 결과(2026-06): GitHub Actions(해외 IP)는 KIS 토큰 발급이 ConnectTimeout으로 차단돼 탈락, NCP(네이버 클라우드) 서울 상시 서버는 실전·모의 토큰 발급 모두 HTTP 200 통과 → 현 단계 호스트 = NCP 서울 상시 서버로 확정.** (Oracle Cloud는 가입 자체가 막혀 미시도.)
- **0-A.3 백테스트 데이터 가용성 게이트 (P0-2) — *차단 게이트***: **과거 5~10년 일봉 + 투자자별 수급 + 공매도 + *상장폐지·거래정지 종목 포함***을 받을 수 있는지 확인한다. 안 되면 워크포워드·시드·생존편향 차단(10장)·켈리 시드(5-2)가 전부 불가. **실행 결과(2026-06): 일봉=KIS 12년+✅, 공매도=KIS 과거(2018치)✅, 과거 수급=KIS 30일만이라 네이버 금융 frgn(외국인·기관 8년+)✅, 상폐/과거시세=네이버 금융 sise_day(한진해운 상폐직전까지)✅. KRX 정보데이터시스템·pykrx는 로그인 차단/노후로 폐기(external-apis §1). → 무료 데이터원만으로 게이트 통과(유료 불필요).**
- **0-A.4 ORD_DVSN 22 스톱지정가 모의 작동 확인 (8장)**: 모의 App Key로 작은 주문 1건(`ORD_DVSN=22` + `CNDT_PRIC`)을 송출해 *모의에서 정상 접수되는지* 확인. 손절 메커니즘이 모의에서 검증 가능한 영역인지 *기능/전략 합격증 경계*(5.2)를 여기서 1차 확정.
- **게이트**: 0-A.2·0-A.3이 "가능"(또는 대안 확정)이어야 0-B로. "불가"인데 대안도 없으면 *설계 전제 자체*(호스트·데이터)를 재검토.

**0-B. 스캐폴딩 + 운영 신뢰성 1차**

- 리포 구조, `config`(모의/실전 플래그·리스크 파라미터·날짜별 세율·모델 단가·rate limit), 시크릿 로딩 단일 진입점(`pydantic-settings`+`tomllib`), **SQLite 스키마 생성**(6장 객체 카탈로그 단일 소스 — `cycles`·`trades`·`outcomes`·`decisions`·`agent_predictions`·`calibration` 뷰 등).
- **운영 신뢰성 1차 박기 (11-2.16 우선순위)**: 11-2.1 사이클 idempotency(`cycles` 상태머신·`recover_pending_cycles()`), 11-2.2 SQLite WAL/busy_timeout/`memory/db.py` 단일 진입점 + 직접 `sqlite3.connect()` 금지 `ruff` 룰, 11-2.5 시크릿 저장소·환경변수만, 11-2.7 시간대 정규화 경계 함수·CI 룰, 11-2.9 "if 실전 분기 금지" `ruff` 룰.
- **결정 사항 동시 확정 (0-A.2 결과 반영)**: 첫 배포 환경(11-2.12), 시크릿 저장소·백업 도구·알림 채널(Discord)·외부 모니터링 서비스(14장 열린 항목).
- **LLM 운영 정책 스캐폴드**: 아직 LLM 호출은 없으나 *호출 어댑터 정책*(11-3.1 재시도·백오프, 11-3.4 `cache_control` 경계)을 미리 박는다.
- **게이트**: 빈 사이클(데이터·LLM 없이 상태머신만)이 idempotency·WAL·시간대 정규화 단위 테스트를 통과.

### Phase 1 — KIS 연동 (모의)

목표: 모의 환경에서 인증·조회·주문·체결기록의 *기본 왕복*을 완성하고, KIS 운영 현실을 결정론 정책으로 흡수한다.

- 인증/토큰 자동 갱신(만료 5분 전 재발급·실패 시 백오프 3회→SafeStop), 잔고·시세·미체결·체결 조회, 모의 주문 송출 — `broker/kis_client.py`.
- **ORD_DVSN 22 본격 검증**: 0-A.4의 sanity를 넘어, 스톱 등록→트리거→자동 체결 흐름과 정정취소 API(`TTTC0013U`) 동작 확인.
- KIS 운영 정책(11-2.3): rate limiter(초당 한도 80%), 5xx 종류별 처리, **주문 송출 실패 시 재시도 금지 + 사이클 내 즉시 체결조회(P1-4)**로 맨몸 포지션 갭 차단.
- **모드 갭 측정 착수**: 동일 작은 주문을 모의에 송출해 체결가·슬리피지 기준선 기록(11-2.15 분기 점검의 1차 데이터; 실전 비교는 Phase 7.5).
- **운영 신뢰성 2차 일부**: KIS 운영·heartbeat·백업(11-2.16 2차).
- **게이트**: 모의계좌로 *인증→시세→모의주문→체결기록 1사이클 수동 실행*(10-2 ②)이 끝까지 돌고, ORD_DVSN 22 작동 여부가 *기능/전략 합격증 경계*로 분류됨.

### Phase 2 — 데이터 레이어

목표: 코드가 데이터를 *수집·보정·압축·라우팅*해 결정 파이프라인이 받을 형태로 정돈한다(통째로 LLM에 안 던진다).

- **후보 선별(스크리닝)** — `data/screener.py`: 제외·거래현실성 필터 → *필터 통과 집합 내* 다섯 항목 백분위 가중 합산 → 상위 40 + 보유. **모집단 = 필터 통과 집합으로 확정(P1-7)**.
- **Tier0** — `data/market_data.py`: 시세·지표(MA·RSI·ATR 등)·투자자별 수급·공매도/대차/신용/프로그램매매·섹터 상대강도·시장 레짐.
- **Tier1 글로벌·매크로** — `data/global_macro.py`: 1a/1b 고정 + 1c 동적 라우팅 + ADR. **시리즈별 백업 가용성 매트릭스 + ADR/美개별주는 KIS 해외주식 시세 1차 대체 검토(P1-7)**.
- **Tier2 이벤트 캘린더** — `data/calendar.py`: 실적·FOMC·금통위·옵션만기·지수 리밸런싱·락업 해제(코드 플래그).
- **데이터 보정**: 수정주가, 거래일/휴장 판정(`exchange_calendars` + KIS 영업일 2중), **데이터 freshness를 타임스탬프 기준으로 점검**(DST·지연 대비), 비용/세금/슬리피지 단일 모델, 백업 출처 폴백(11-2.18), vintage 처리(4.2).
- **뉴스 헤드라인 수집** — 네이버 검색 API thin wrapper: **종목 매칭 노이즈 처리(종목코드/섹터 조합 쿼리·발행시각 윈도우·오매칭 제거, P1-7)**. (판단은 Phase 4 LLM.)
- **게이트**: Tier0/1/2 수집·압축 결과 정확성 + 라우팅이 의도대로 + vintage 정책 작동(10-2 ③).

### Phase 3 — 리스크 엔진 + 사이징·청산·포트폴리오 (결정 파이프라인보다 먼저)

목표: LLM 제안을 검증·집행할 *결정론 바닥*을 완성한다. 전부 코드, LLM 위임 없음.

- **하드룰·서킷브레이커** — `risk/risk_engine.py`: 절대 바닥(종목당 ≈13%·섹터 30%·총노출 100%·**일일 −4% — 측정 정의는 MTM 포함·당일시작자본 대비·사이클 스냅샷, P1-1**), 검사 순서·충돌 처리(A.1), 재개·복구 절차(A.2), 모델 이상행동 임계(A.3), 한국시장 특화(점상/VI/단기과열/배당락).
- **사이징** — `risk/sizing.py`: **변동성 타깃팅 = 메인 엔진(1일차 작동) + 프랙셔널 켈리 = 자동활성 상한(관련 칸 n≥20일 때만, 그 전 휴면), 최종수량 = min(변동성, 켈리천장, 하드한도), 켈리 입력 p=calibration·b=outcomes·시드 활용 (P0-3)**. **소수종목 시 종목당 한도 자동 상향(P0-3 ④)**.
- **청산** — `exec/exits.py`: **R = 진입시점 \|진입가−최초손절가\| 영구고정(P1-2)**, 우선순위(논지무효→손절→+1.5R 부분익절→트레일링→시간), **집행 채널 선후 = 선행게이트 동기화 후 잔여보유에만 적용(이중주문 방지, P1-2)**, 스톱 에스컬레이션. tp1_R·trail_k·max_hold_days는 config 시작값(백테스트 확정).
- **포트폴리오** — `risk/portfolio.py`: 동시보유 목표(자본 연속함수), 섹터 분산, 교체매매, 레짐별 노출.
- **게이트**: 리스크 엔진 룰별 단위 테스트(10-2 ①) + 사이징 켈리 활성/휴면 경계 테스트(5-2 검증 a) 통과.

### Phase 4 — 결정 파이프라인 (린 LLM)

목표: 코드가 못 하는 두 곳(뉴스 글 판단·최종 결정)에만 LLM을 끼워 9단계 사이클을 완성한다.

- **뉴스·촉매 분석가** — `agents/catalyst.py`: 워치리스트 헤드라인·펀더멘털을 *묶음 1회* 호출(헤드라인 있는 종목·상위 K개로 범위 제한), 출력 스키마(`view`·`confidence`·`key_signals`·`key_risks`).
- **결정 에이전트** — `agents/decider.py`: 숫자점수+촉매+교훈+보정통계+리스크제약+보유현황 → JSON 제안주문(신규+보유 동적관리). 반대 시나리오는 지시문에 포함(별도 호출 없음).
- **B안 코드 결정 규칙 병행 구현 (P1-3)**: Phase 6 3자 비교(A/B/C)를 위해, *결정 에이전트 없이 종합점수 임계로 결정하는 B 경로*를 함께 구현해 둔다(C와의 차이를 결정 주체 하나로 고립). B는 이 시스템의 *가장 싸고 단순한 완결 버전*이자 검증 대조군.
- 9단계 파이프라인 조립 — `pipeline/trading_cycle.py`(단일 진입점 `run_trading_cycle.py`). JSON 검증/4단계 폴백(11-3.3), 부분 실패 룰(11-3.5).
- **게이트**: 실주문 차단 플래그로 *파이프라인 드라이런*(10-2 ④) — 뉴스 견해·결정 JSON·반대 시나리오 반영·로그 검증.

### Phase 5 — 기억·학습 루프 (정량, 코드 중심)

목표: 돈에 영향을 주는 학습을 *전부 코드(SQL/numpy)*로 닫는다. 회고(자연어 교훈)는 미도입.

- 저널 기록·SQL 검색 — `memory/journal.py`.
- **보정통계** `calibration`(에이전트×레짐×확신도 — 세션·트리거 차원은 표본 차면 자동) + 수축·최소표본·신뢰구간 3종 세트(7.18).
- **반사실** `shadow_outcomes`(거부·무거래 가상손익, 가중 ≤0.3).
- **인출 스코어러**(레짐·섹터·셋업·최근성·보정 가중). **회고 휴면 기간 공백 메우기 = lessons 없을 때 decisions↔outcomes 원시 구조화 매칭 인출(P1-6)**. 임베딩/RAG 미사용.
- **회고 에이전트는 만들지 않음** — `helped_score`·인출품질(10-2 #11)로 *교훈이 비용 떼고도 도움 됨*이 확인될 때만 후속 추가.
- **게이트**: 정량 학습 루프가 코드로 도는지 확인(10-2 #10) — 보정 집계·적재·인출 우선순위·소표본 제외.

### Phase 6 — 백테스트·성과평가 하니스 (모의 검증보다 먼저)

목표: *전략이 비용·세금·LLM비를 다 빼고도 단순전략을 이기는지*를 과거로 증명한다. 수익 진위 검증의 토대.

- 과거 재생 엔진 — `backtest/engine.py`: 거래세·수수료·슬리피지·체결현실 반영, **룩어헤드·생존편향 차단(상폐·거래정지 포함 — Phase 0-A.3 데이터 전제)**.
- 워크포워드 OOS 분리 — `backtest/walkforward.py`. 4종 벤치마크(코스피 매수후보유·단순 모멘텀·현금·**균등가중 워치리스트**).
- **3자 비교 (A 코드만 / B 뉴스만 / C 뉴스+결정)** — 10-3.1 절차(A에서 파라미터 확정, 뉴스 출력 캐싱, 결정자 소수회). C≈B면 `decider.py` 제거.
- KPI 3겹(세전/세후/net) — `eval/metrics.py`. 과최적화 방지(파라미터 7개·민감도·**PBO·Deflated Sharpe**) — `eval/gate.py`.
- **학습 테이블 시드**: 5~10년 재생으로 calibration·outcomes 칸을 선채움(시간가중 자동 적용). 켈리 p·b 초기값·confidence 0.5배 정책(7.19).
- **게이트 (★실자본 경로의 1차 관문)**: **방향성 Go/No-Go 통과(P1-5)** — net 기준 4벤치마크 전부 초과 + Deflated Sharpe>0 + PBO<50% + 민감도 절벽 없음. *하나라도 미달이면 No-Go*(절대 임계는 자금 확정 후 추가).

### Phase 7 — 모의투자 *기능* 검증

목표: 전 파이프라인이 무인으로 *끝까지 도는지*를 확인. **기능 합격증만** 발급(전략 합격 아님).

- 스케줄러 연결(NCP 서버 cron), 검증 기간 무인 운영, 9단계가 끝까지 도는지.
- 확인 대상: 파이프라인 흐름·리스크 룰 동작·LLM 스키마 통과/실패 거부·SQLite 무결성·잔고 정합성 게이트·KIS 토큰/rate limit·cron heartbeat·시간대 정규화·로그 회전.
- **운영 신뢰성 3·4차**(11-2.16): "if 실전" 분기 CI·마이그레이션·주문 유형 정책표·**분기 운영 점검 체크리스트 가동**.
- **게이트**: *기능 합격(이진 통과/미통과)*. Phase 7 KPI는 *모드 갭 패널티 할인 후*가 아니면 Go/No-Go에 직접 입력 금지(부풀려진 값).

### Phase 7.5 — 소액 실거래 *전략* 검증

목표: KIS 모의가 *못 주는* 합격증을 실자본 극소액으로 측정한다.

- **전제**: Phase 6 OOS 방향성 게이트 통과 + Phase 7 기능 합격.
- 자본의 5~10%(ADV 1% 통과 범위)로 *실거래 모드* 1~3개월.
- 측정: **체결 메커니즘 실측이 주(主)** — 실제 슬리피지·미체결률·체결 품질·VI/상하한가 락업 실동작·ORD_DVSN 22 실작동·모드 갭. **수익 KPI(Sharpe·MDD·실현손익분포·calibration 실적중)는 표본이 작아(30~120건) *신뢰구간만* 보고 점추정으로 단정하지 않는다**.
- **게이트**: *전략 합격증* — 실측 체결 품질이 백테스트 가정과 크게 어긋나지 않고, 운영 알림(silent failure 감지)이 실효. 결과가 Phase 10 본격 실거래 게이트의 1차 입력.

### Phase 8 — 대시보드

목표: 매매와 분리된 *읽기 전용* 사후 확인·복기 도구.

- Streamlit + plotly — `dashboard/app.py`: 5영역(나의 정보·수익곡선+벤치마크 오버레이·리포트 드릴다운·LLM 사용량·오류/알림). 읽기 전용 연결(불변식 8.1), 자동 새로고침.
- **모드 갭 탭**: Phase 7 모의·7.5 소액·10 실거래 KPI를 별도 탭으로 비교(모의가 실전 대비 얼마나 낙관적인지). 스냅샷 신선도 + *마지막 성공 사이클 vs 현재 시각 갭* 경고(silent failure 가시화).
- 접속: NCP 서버 대시보드를 SSH 터널/Tailscale로(현). 외부 공개 노출 0(자본시장법 게이트 11-2.17).
- **게이트**: 5영역이 SQLite 소스와 정확히 매핑되고 읽기 전용 불변식 위반 없음.

목표: 확보·검증된 데이터원만 추가(데이터 없는 전문가는 환각).

- Tier3a **DART 공시 *본문* 분석가**(`OpenDartReader`·저작권 정리 후) — 4.5절 ②. vintage 처리(수신 시점 스냅샷·정정/취소 라벨) 동일 적용.

### Phase 10 — 본격 실거래 + 단계적 자본 램프업

목표: 검증된 전략에만, 검증된 비중으로, 단계적으로 자본을 늘린다.

- **진입 조건**: Phase 6 OOS 통과 + Phase 7.5 전략 합격증 + 모드 갭 반영 후 KPI가 *절대* Go 임계 충족(자금 확정 후 14장에서 확정). *Phase 7 모의 KPI는 직접 입력하지 않음*(통과/미통과 이진만).
- **램프업**: 한 번에 100% 아님 — 예) 10%(=Phase 7.5) → 30% → 60% → 100%, 각 단계 1~2개월 운영 후 *KPI가 직전 단계 대비 악화되지 않은 경우에만* 다음 단계. 악화 시 직전 비중으로 되돌리고 원인 분석.
- 외부 이중 안전망(5-1 C): KIS 출금한도 0·일일 거래한도 설정·실시간 알림. 실거래 전환 후 첫 30일 매일 수기 점검.
- **게이트**: 각 램프업 단계가 *직전 단계 대비 KPI 비악화*. 어느 단계든 측정상 유의미 악화 시 자본 회수.

### 12.1 의존성 한눈에 (진행 흐름)

```
Phase 0 [선검증 게이트 → 스캐폴딩]
   │  (호스트·데이터 가용성 확정되어야 그 아래 전부 의미)
   ▼
Phase 1 [KIS 모의] → Phase 2 [데이터] → Phase 3 [리스크/사이징/청산]
                                              │ (결정론 바닥 먼저)
                                              ▼
                                        Phase 4 [결정 파이프라인 + B안]
                                              ▼
                                        Phase 5 [학습 루프(코드)]
                                              ▼
                                        Phase 6 [백테스트] ──★방향성 Go/No-Go★
                                              ▼ (통과해야 실자본 경로)
                                        Phase 7 [모의·기능합격]
                                              ▼
                                        Phase 7.5 [소액실거래·전략합격]
                                              ▼
                                        Phase 10 [본격실거래·램프업]

Phase 8 [대시보드]  ─ Phase 6 이후 언제든(읽기 전용, 매매와 독립)

```
